Terminal Access Controller Access Control System (TACACS)

A protocol for providing centralized access control for users who access network services.

Description

Terminal Access Controller Access Control System (TACACS) is a protocol developed for handling authentication, authorization, and accounting (AAA) for users who access networked devices. Originally developed by BBN Technologies for ARPANET, TACACS allows for centralized authentication of users attempting to access system resources, enabling organizations to manage user permissions more efficiently and securely. TACACS is often used in conjunction with routers, switches, and other network devices, allowing for secure management of user credentials. The protocol works by having users send their credentials to a TACACS server, which then validates these credentials and communicates the access permissions back to the requesting device. There are different versions of TACACS, including TACACS+ which provides enhanced security features such as encryption of the entire payload, making it more secure than its predecessors. With its ability to manage user sessions and track user activity, TACACS is a vital component in Identity and Access Management (IAM) systems, ensuring that only authorized personnel have access to critical network resources.

Examples

  • Used by network administrators to authenticate user access on Cisco routers and switches.
  • Implemented in enterprise environments for secure management of network infrastructure.

Additional Information

  • TACACS+ is the most widely used version due to its enhanced security features.
  • TACACS can be integrated with other IAM solutions for comprehensive access management.
