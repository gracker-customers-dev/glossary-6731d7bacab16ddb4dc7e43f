Distinguished Name (DN)

A unique identifier for an object in a directory service, such as LDAP.

Description

A Distinguished Name (DN) is a string representation of the path to a specific object within a directory service. It is commonly used in Lightweight Directory Access Protocol (LDAP) and other directory services to uniquely identify users, groups, and other entities. The DN is composed of a sequence of relative distinguished names (RDNs) that represent the object's position in the directory hierarchy. Each RDN consists of an attribute name and a corresponding value, separated by an equal sign. DNs are essential for managing access control and authentication in Identity and Access Management (IAM) systems, as they provide a way to reference and interact with directory objects programmatically. A well-structured DN helps ensure that each entry is uniquely identifiable, which is crucial for maintaining security and integrity in directory services.

Examples

  • cn=John Doe,ou=Users,dc=example,dc=com
  • uid=jdoe,ou=People,dc=example,dc=com

Additional Information

  • DNs are case-sensitive and must be formatted correctly to ensure proper identification.
  • The structure of a DN can vary based on the directory service and its schema.
