Relative Distinguish Name (RDN)

A Relative Distinguished Name (RDN) is a component of a Distinguished Name (DN) in a directory service, used to uniquely identify an entry within a specific context.

Description

In the context of directory services such as LDAP (Lightweight Directory Access Protocol), a Relative Distinguished Name (RDN) is a unique identifier for an object within its immediate parent context. The RDN consists of one or more attribute-value pairs, where the attribute is typically a common name (CN), organizational unit (OU), or similar identifier. The RDN is crucial for distinguishing objects that may reside within the same organizational unit or container. For example, in a user directory, an RDN could be 'cn=John Doe' for a user named John Doe within an organizational unit. The RDN is part of a larger Distinguished Name (DN) which provides a complete path to the object by including additional context information. RDNs help maintain hierarchy and organization within directory structures, making it easier to locate and manage entries. Properly defining RDNs is essential for efficient directory queries and for implementing access control and identity management.

Examples

  • RDN example: 'cn=Jane Smith' for a user object named Jane Smith.
  • RDN example: 'ou=Sales' for an organizational unit containing sales department users.

Additional Information

  • RDNs can be composed of multiple attributes, such as 'cn=John Doe, ou=Employees'.
  • RDNs must be unique within their parent context to avoid ambiguity.
