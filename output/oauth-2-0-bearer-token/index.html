OAuth 2.0 Bearer Token

A type of access token used in the OAuth 2.0 authorization framework that allows access to a protected resource.

Description

OAuth 2.0 Bearer Token is a type of access token that is issued by an authorization server in the OAuth 2.0 framework. This token is used to gain access to protected resources on behalf of the user or client application. The term 'Bearer' indicates that possession of the token is sufficient to gain access to the resource; no additional authentication is required. Bearer tokens are typically included in HTTP requests as an 'Authorization' header. For example, a request might look like: 'Authorization: Bearer '. The security of the Bearer token is reliant on its confidentiality, meaning it should be transmitted securely (e.g., using HTTPS) to prevent interception by unauthorized parties. Bearer tokens can have expiration times and may need to be refreshed periodically, depending on the implementation. They are widely used in modern web applications, APIs, and services to enable secure and delegated access.

  • They are often used in conjunction with refresh tokens to enhance security and usability.
