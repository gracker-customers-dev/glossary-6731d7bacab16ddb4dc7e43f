Identity Token (id_token)

A security token that contains identity information about a user and is issued by an identity provider.

Description

An Identity Token, commonly referred to as an id_token, is a JSON Web Token (JWT) that is issued by an identity provider (IdP) as part of the authentication process. It serves to convey information about the authenticated user, including their identity and any relevant claims such as email address, roles, and permissions. The id_token is typically provided as part of the OpenID Connect protocol, enabling client applications to verify the user's identity without directly handling sensitive credentials. The token is signed by the IdP to ensure its integrity and authenticity, allowing the receiving application to trust the information contained within it. In many use cases, the id_token is used alongside access tokens, which grant permissions to access resources. The id_token is crucial for supporting Single Sign-On (SSO) scenarios, where users can authenticate once and gain access to multiple applications without needing to log in separately to each one.

Examples

  • An id_token issued after a user successfully logs in via Google Sign-In.
  • An id_token received in response to a successful authentication request in an application using Azure Active Directory.

Additional Information

  • id_tokens are typically short-lived and may have an expiration time.
  • They can include scopes, which define the level of access or permissions granted to the user.
