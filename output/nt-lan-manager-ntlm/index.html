NT Lan Manager (NTLM)

NT Lan Manager (NTLM) is a Microsoft authentication protocol used in network environments to authenticate users and services.

Description

NT Lan Manager (NTLM) is a suite of Microsoft security protocols that provides authentication, integrity, and confidentiality to users and services on Windows networks. It was originally developed for Windows NT and has been used in various versions of Windows operating systems. NTLM operates through a challenge-response mechanism, where the server sends a challenge to the client, which then responds with a hashed version of the password combined with the challenge. This process helps authenticate the user without transferring the actual password over the network. While NTLM has been widely used, it has several vulnerabilities, including susceptibility to pass-the-hash attacks and replay attacks. As a result, Microsoft recommends using more secure protocols, such as Kerberos, especially in environments where sensitive information is handled. Despite its vulnerabilities, NTLM remains in use for backward compatibility with older systems and applications that do not support newer authentication protocols.

Examples

  • Used in Windows NT and early versions of Windows networks for user authentication.
  • Still supported for legacy applications that require NTLM authentication.

Additional Information

  • NTLM has been largely replaced by Kerberos in modern Windows environments.
  • Organizations are encouraged to migrate to more secure authentication methods to enhance security.
