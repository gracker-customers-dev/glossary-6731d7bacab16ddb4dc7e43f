Message Queuing Telemetry Transport (MQTT)

A lightweight messaging protocol for small sensors and mobile devices optimized for high-latency or unreliable networks.

Description

Message Queuing Telemetry Transport (MQTT) is a publish-subscribe based messaging protocol that is designed for lightweight communication between devices in constrained environments. Originally developed by IBM in the late 1990s, it is now widely used in Internet of Things (IoT) applications where bandwidth and battery power are at a premium. MQTT operates on top of the TCP/IP protocol, providing a simple and efficient way to send messages between devices, facilitating real-time data exchange. The protocol supports three quality of service levels, allowing developers to balance between message delivery guarantees and network efficiency. Its minimal overhead and small code footprint make it particularly suitable for devices with limited processing power and memory. MQTT uses a broker to manage message distribution, enabling decoupling of message producers and consumers, which enhances scalability. This makes it an ideal choice for applications such as home automation, telemetry, and monitoring systems, where devices need to communicate reliably over possibly unreliable networks.

Examples

  • Smart home devices communicating sensor data to a central hub.
  • Industrial equipment sending telemetry data to cloud-based analytics platforms.

Additional Information

  • MQTT is standardized by OASIS and ISO/IEC.
  • The protocol is widely supported by various programming languages and IoT platforms.
