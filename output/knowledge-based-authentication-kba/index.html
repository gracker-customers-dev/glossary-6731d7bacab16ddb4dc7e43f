Knowledge-based authentication (KBA)

A security process that involves verifying a user's identity by asking them to provide information only they should know.

Description

Knowledge-based authentication (KBA) is a method used in identity and access management (IAM) to verify a user's identity by requiring them to answer specific questions that only they should know the answers to. KBA can be categorized into two types: static and dynamic. Static KBA involves pre-set questions, such as a user's mother's maiden name or the name of their first pet, which are established during an account setup process. Dynamic KBA, on the other hand, generates questions based on user data, such as previous transactions or account activity, that change over time. While KBA can enhance security by adding an extra layer of verification, it is important to note that it can be vulnerable to social engineering attacks and data breaches. To mitigate these risks, organizations often implement KBA alongside other authentication methods, such as two-factor authentication (2FA), to create a more robust security framework.

Examples

  • Answering security questions during account recovery.
  • Verifying identity by providing information from past transactions.

Additional Information

  • Static KBA is more vulnerable to breaches than dynamic KBA.
  • KBA is often used in conjunction with other authentication methods for enhanced security.
